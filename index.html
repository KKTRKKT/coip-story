프로젝트 개요

Overview

암호화폐 시장의 높은 변동성과 진입 장벽으로 초보 투자자가 실제 자금 손실 없이 매매 경험을 쌓기 어렵습니다. COIP는 실시간 시세는 그대로 사용하되 주문·체결·잔고만 서버 내 시뮬레이션으로 처리해, "현실과 동일한 결과"를 약속하는 모의투자 환경을 제공합니다.

◆ 핵심 가치

  • 리스크 제로 연습 — 가상 KRW 1천만 원 초기 지급
  • 실시간 시세 — 업비트 KRW 마켓 실가격 사용 (260종목)
  • 성과 추적 — 누적 손익·수익률·거래내역으로 실력 확인
  • 트레이더 커뮤니티 — 분석·전략·인증을 게시글·채팅으로 공유

▣ 타겟 사용자

  • 주 타겟 — 실제 자금 투입이 부담스러운 초보 투자자
  • 부 타겟 — 새로운 매매 전략을 검증하려는 중급 투자자
  • 데스크탑 우선 — 최소 1280px, 한국어·KRW 기준
  • 모의투자 원칙 — 실제 자산·입출금·레버리지 없음

시스템 아키텍처

Architecture v0.6

"DTL이 기본, REST는 필요한 곳만" 원칙으로 단순함을 우선했습니다. 시세 데이터는 백엔드를 거치지 않고 클라이언트가 Upbit API에 직접 연결해 복잡도를 줄였습니다. -> 사용자 많아지면 이용약관 위배이므로 수정 예정

▸ Application Flow

Browser │ ├── HTTP ─▶ Django (WSGI) │ ├── DTL 뷰 (커뮤니티·마이페이지) │ ├── ModelForm + messages │ └── DRF (시세·주문 JSON) │ ├── WebSocket ─▶ Daphne (ASGI) │ └── ChatConsumer (GLOBAL · 코인별) │ └── 직접 호출 ─▶ Upbit Public API ├── REST /v1/ticker, /v1/candles └── WebSocket 실시간 체결가·호가

▸ Data & Infrastructure

Django Backend │ ├──▶ pgbouncer:5432 (transaction) │ └──▶ PostgreSQL (StatefulSet, 20Gi) │ ├──▶ Redis (Channel layer · group_send) │ └──▶ MEDIA_ROOT /avatars/ (NFS RWX) External Services: ├── SMTP (이메일 인증) └── Google OAuth 2.0
